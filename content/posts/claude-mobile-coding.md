## 背景：Claude 移动端的 Codes 功能

最近，Claude 手机端 App 更新了一个名为 **"Codes"** 的功能。虽然 PC 端同样具备，但对移动端来说，这次更新带来的变革意义更大——**你真的可以用手机写代码，并且实时推送到线上了。**

这不是概念演示，而是我正在使用的真实工作流。这篇文章本身，就是我用 Claude 手机端写完并推送上线的。

## 核心能力

### 1. 连接 GitHub 仓库

Codes 功能可以直接连接你的 GitHub 仓库，并在云端建立一个完整的开发环境。不需要在手机上安装任何 IDE 或终端工具，Claude 会帮你处理一切。

### 2. 对话式编程

连接仓库后，你可以直接通过自然语言对话来写代码。告诉 Claude 你想要什么，它会在云环境中编写、修改、调试代码。整个过程就像和一个程序员搭档聊天一样自然。

### 3. 实时推送上线

结合 GitHub Personal Access Token，Claude 可以将代码实时推送到 GitHub 主干分支。如果你的项目配置了自动部署（比如 GitHub Pages、Vercel、Netlify 等），推送即上线，代码写完几秒钟就能在线上看到效果。

## 实际应用场景

### 博客更新

如果你的博客部署在 GitHub Pages 等平台上，这个功能堪称完美搭档。你可以随时随地让 Claude 帮你写一篇文章，写完直接推送，博客即刻更新。不需要打开电脑，不需要 SSH 连接服务器，一切在手机上完成。

### 个人网页与 Web App

对于部署在线上的个人网页或 Web 应用，同样适用。需要改个样式？加个功能？修个 Bug？掏出手机，描述你的需求，Claude 帮你改完直接推上去。

### 轻量级运维与修复

线上出了个小问题，手边又没有电脑？打开 Claude，连上仓库，描述问题，让它帮你定位并修复，然后推送上线。整个过程可能只需要几分钟。

## 我的真实感受

### 随时随地的生产力

这种体验最让我兴奋的一点是：**编程不再被办公桌束缚了。** 通勤路上、咖啡店里、甚至躺在沙发上，只要有想法，就可以立刻动手实现。从"想到"到"上线"的路径被压缩到了极致。

### 开发门槛的进一步降低

AI 编程本身已经在降低开发门槛，而移动端的加入让这件事更进一步。你不需要配置本地开发环境，不需要记住复杂的命令，只需要清晰地描述你的需求。这意味着更多人可以参与到软件开发中来。

### 工作流的根本改变

以前的工作模式是：坐在电脑前，打开好几个窗口，IDE、终端、浏览器来回切换，一守就是一整天。

现在的工作模式变成了：**随时随地发一条指令，让 AI 去执行，执行完了通知我来检查和确认。** 我不再需要全程盯着屏幕，而是可以把精力放在思考和决策上，把执行交给 AI。

这不只是效率的提升，更是一种工作方式的根本转变。

## 写在最后

Claude Mobile Codes 让"用手机写代码"这件事从玩具变成了真正的生产力工具。它不会取代 PC 端的深度开发，但在很多场景下，它已经足够好用了。

如果你也有 GitHub 上的项目，不妨试试这个功能。也许你会像我一样发现：**原来写代码这件事，可以这么轻松。**
